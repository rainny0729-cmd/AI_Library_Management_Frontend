# 河南理工大学 AI 图书馆综合管理与知识服务平台

## 项目背景

本项目是面向校内展示、AI token 额度申请、项目答辩和信息化成果汇报的高保真网页端工程。系统定位为河南理工大学图书馆的 AI 综合管理与知识服务平台，覆盖馆藏知识库、图书借阅、座位预约、入馆流量、学习问答、推荐系统、多 Agent 调度和管理决策等场景。

系统面向河南理工大学约 3.5 万名学生的图书馆学习与管理场景，因涉及校内业务数据、借阅记录、座位预约和内部管理接口，当前仅用于校内环境展示与部署，未上架 GitHub。

## 核心功能

- AI 馆藏知识库：自然语言检索、RAG 召回、引用来源、知识图谱与索引更新记录。
- 管理员数据指挥舱：入馆、借阅、座位、AI 问答、任务队列、实时日志与管理建议。
- 图书管理：馆藏台账、高级筛选、状态标记、AI 推荐指数与馆藏优化建议。
- 借阅管理：借还趋势、用户画像、风险提醒与自动生成的周度借阅摘要。
- 座位预约：按楼层热力图、时间段切换、座位状态和压力预测。
- 入馆流量分析：出入口统计、小时流量、学院分布、楼层热度和异常检测。
- 学生端：我的借阅、我的预约、推荐书单、AI 学习助手、学习路线和个人知识库收藏。
- Agent 调度中心：12 类 Agent 的调用量、token、成功率、队列、延迟和协作链路。
- token 消耗说明：月度额度测算、消耗来源、请求趋势和额度申请说明。
- 系统日志与审计：API 调用、知识库更新、Agent 调度、脱敏状态、权限检查和模型输出审计。

## 技术栈

- React 18
- TypeScript
- Vite
- Tailwind CSS
- lucide-react
- recharts
- framer-motion

## 页面模块

- `/home` 系统首页 / 项目总览
- `/dashboard` 管理员数据指挥舱
- `/knowledge` AI 知识库
- `/books` 图书管理
- `/borrow` 借阅管理
- `/seats` 座位预约
- `/traffic` 入馆流量分析
- `/qa` 学习问答
- `/recommend` 推荐系统
- `/purchase` 采购建议
- `/agents` Agent 调度中心
- `/tokens` token 消耗与额度申请说明
- `/logs` 系统日志与审计
- `/student` 学生端页面

项目使用 hash 路由，例如 `http://localhost:5173/#/dashboard`。

## 本地运行方式

```bash
npm install
npm run dev
```

构建检查：

```bash
npm run build
```

## 数据说明

当前使用脱敏模拟数据，集中维护在 `src/data/mock.ts`。数据覆盖学生规模、馆藏数量、借阅记录、座位状态、入馆流量、热门图书、Agent 状态、token 消耗、系统日志、知识库更新记录、推荐书单、学院分布、楼层分布和时间段数据。

后续接入真实服务时，可将 `src/data/mock.ts` 替换为 API 请求层，页面和组件结构已按业务模块拆分，便于对接图书馆管理接口、统一身份认证、知识库服务和 Agent 调度服务。

## 部署说明

项目为标准 Vite 静态前端工程，可执行 `npm run build` 生成 `dist/` 目录，再部署到校内服务器、Nginx、对象存储静态站点或信息化平台统一前端网关。
